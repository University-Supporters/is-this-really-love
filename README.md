# Is This Really Love? 💘 (이게 정말 사랑일까?)

> **"지연이... 아니면 내 사랑❤️? 당신의 전화기로 걸려오는 집착의 목소리를 직접 체험해보세요."**

본 프로젝트는 텍스트 중심의 기존 인권 교육에서 벗어나, 실제 상황과 유사한 **체험형 웹 시뮬레이션**을 통해 스토킹 및 교제폭력의 위험성을 알리고자 제작되었습니다. 사용자분들께서는 브라우저 상에서 직접 '집착'과 '통제'의 상황을 마주하며, 폭력에 대한 인식을 새롭게 정립(Awareness Transition)하실 수 있습니다.

---

## 🚀 주요 기능 (Main Features)

### 1. 사실적인 수신 전화 시뮬레이션 📞
- **리얼한 UI 및 진동 효과**: iOS 환경을 충실히 재현한 수신 화면과 브라우저 진동 API(Vibration API)를 활용하여, 실제 전화를 받는 듯한 긴장감 넘치는 경험을 제공합니다.
- **배경 이미지 연동**: 전화 수신 시 발신자의 사진이 배경에 표시되어, 일상을 파고드는 통제와 집착의 상황을 시각적으로 몰입감 있게 전달합니다.

### 2. 성별 맞춤형 랜덤 발신자 시스템 👥
- **성별 최적화 시나리오**: 사용자의 성별 선택에 따라 각기 다른 관점의 시뮬레이션이 진행됩니다.
- **다양한 사례 체험**: 각 성별 버전별로 5명씩, 총 10개의 랜덤 발신자 시나리오가 준비되어 있습니다. 매번 새로운 사례를 통해 폭력의 다양한 양상을 확인하실 수 있습니다.

### 3. 몰입도 높은 통화 인터페이스 🎧
- **실제 녹음본 재생**: 통화 수락 시 실제 상황을 가정한 음성 녹음본이 재생되어 상황의 심각성을 생생하게 전달합니다.
- **실시간 애니메이션**: 통화 상태를 보여주는 역동적인 파형(Visualizer)과 타이머를 통해 실제 통화 중인 듯한 환경을 구성하였습니다.

### 4. 실감나는 통화 종료 연출 및 트랜지션 ⚡
- **글리치 효과(Glitch Effect)**: 통화 종료 시 'Connection Terminated' 문구와 함께 화면이 흔들리는 글리치 애니메이션과 붉은색 맥동 효과를 적용하여 강렬한 인상을 남깁니다.
- **사운드 디자인**: Web Audio API를 활용한 저역대 노이즈 사운드를 통해 통화가 단절되는 순간의 긴장감을 청각적으로 구현했습니다.

### 5. 강화된 피해 지원 정보 및 가이드 🛡️
- **법적 처벌 안내**: 스토킹처벌법에 따른 징역 및 벌금 규정(반의사불벌죄 폐지 등)을 시각화된 카드로 상세히 안내합니다.
- **긴급 도움처 및 핫라인**: 112, 1366 뿐만 아니라 '경기도젠더폭력통합대응단'의 24시간 핫라인(010-2989-7722)과 마주봄 홈페이지를 즉시 연결할 수 있도록 구성하였습니다.
- **스크롤 애니메이션**: 정보 화면에 섹션별 스크롤 트리거 애니메이션을 도입하여 모바일 환경에서도 전문적인 정보를 몰입감 있게 확인할 수 있습니다.

## 🏗 아키텍처 및 폴더 구조 (Architecture)

v3.0 리팩토링을 통해 단일 파일(App.jsx) 중심의 구조에서 **기능별 모듈 분리** 구조로 전환되었습니다.

```text
is-this-really-love/
├── public/                         # 📁 정적 자산 관리
│   ├── audio/                      # 🔊 시나리오별 실제 음성 녹음 파일
│   └── images/                     # 📸 발신자 프로필 및 배경 이미지
├── src/                            # 🌐 소스 코드
│   ├── constants/
│   │   └── callers.js              # 발신자 데이터 및 외부 링크 상수
│   ├── hooks/
│   │   └── useCallSession.js       # 통화 상태·오디오·진동·타이머 커스텀 훅
│   ├── components/
│   │   ├── icons/
│   │   │   └── index.jsx           # SVG 아이콘 통합 모듈
│   │   └── screens/
│   │       ├── SelectionScreen.jsx # 성별 선택 화면
│   │       ├── IncomingScreen.jsx  # 수신 전화 화면
│   │       ├── InCallScreen.jsx    # 통화 중 화면
│   │       ├── EndingScreen.jsx    # 종료 연출(글리치) 화면
│   │       └── InfoScreen.jsx      # 고도화된 안내 정보 화면
│   ├── App.jsx                     # 화면 라우터 (35줄)
│   ├── index.css                   # 전역 스타일 및 Tailwind 설정
│   └── main.jsx                    # 애플리케이션 진입점
└── tailwind.config.js              # 디자인 시스템 및 애니메이션 설정
```

## ⚙️ 시작하기 (Getting Started)

아래 명령어를 통해 로컬 환경에서 시뮬레이션을 실행해 보실 수 있습니다.

1. 의존성 패키지를 설치합니다.
   ```bash
   npm install
   ```
2. 개발 서버를 실행합니다.
   ```bash
   npm run dev
   ```
3. 브라우저에서 `http://localhost:5173`에 접속하여 체험을 시작합니다.

## 🎮 시뮬레이션 흐름 (Simulation Flow)

1. **성별 선택**: 사용자의 성별을 선택하여 맞춤형 시나리오를 준비합니다.
2. **수신 대기**: "시작하기" 버튼 클릭 시, 가상의 발신자로부터 전화가 걸려오며 진동이 시작됩니다.
3. **상황 대응**: 전화를 수락하거나 거절하며 상황에 대응해 봅니다.
4. **강렬한 엔딩**: 통화 종료 후 발생하는 글리치 연출을 통해 상황의 심각성을 인지합니다.
5. **정보 습득**: 강화된 정보 화면에서 법적 처벌 수위와 전문 상담 기관 정보를 확인합니다.
6. **캠페인 참여**: 공식 SNS를 방문하여 인권 보호 활동에 동참합니다.

### v3.2 (2026.05.06)
- **발신자 시나리오 다변화 및 프로필 풀 확장**:
  - **체험 다양성 강화**: 성별 체험 시나리오 풀을 기존 각 3명(총 6명)에서 **각 5명(총 10명)으로 확장**하여, 더욱 다각화되고 현실감 넘치는 교제폭력 및 집착의 양상을 체험할 수 있도록 고도화하였습니다.
  - **신규 프로필 자산 연동**: 새롭게 추가된 발신자들을 위해 현대적이고 이질감 없는 고품질 프로필 사진 자산(`male_4.jpg`, `male_5.jpg`, `female_4.jpg`, `female_5.jpg`)을 생성 및 연동하였습니다.
  - **중복 선택 방지 및 비복원 랜덤 구현**: 동일 인물이 연속으로 당첨되는 등 단순 무작위 난수의 단점을 보완하고자, 전체 풀이 1회씩 모두 소진되기 전까지 중복을 완전 차단하는 **비복원 랜덤(Shuffle-History Queue)** 제어 방식을 구축하였습니다.
  - **성별 무관 옵션 신설**: 여성/남성 역할에 국한되지 않고 10종의 발신자 연출 전체를 골고루 무작위 체험할 수 있도록 시작 선택 화면에 **'성별 무관' 선택 항목**을 신규 개발하고 레이아웃을 반응형으로 보정했습니다.

### v3.1 (2026.05.06)
- **UI/UX 레이아웃 및 렌더링 버그 수정**:
  - **인스타그램 이동 버튼 테두리 잘림 해결**: 정보 화면의 애니메이션 요소에 적용되었던 제한적 최적화 속성(`contain: content`)을 정밀 조정하여, 마우스 호버 확대 시 버튼 테두리가 잘리는 현상을 해결하였습니다.
  - **경고 아이콘 상단 꼭짓점 깨짐 해결**: 삼각형 경고 SVG 아이콘의 stroke 굵기로 인해 상단부 꼭짓점이 평평하게 깎여 보이던 현상을 `overflow: visible` 스타일 지정을 통해 예리하고 온전하게 렌더링되도록 수정하였습니다.

### v3.0 (2026.05.04)
- **전체 코드 모듈화(리팩토링)**: 단일 `App.jsx`를 기능별로 완전 분리하여 유지보수성을 극대화하였습니다.
- **시퀀스 기반 종료 연출**: 'Connection Terminated' 문구가 즉시 나타난 후, 퇴장 시에만 지직거리는 글리치(Out-glitch) 효과가 발생하는 정밀한 시퀀스 애니메이션을 구현하였습니다.
- **모바일 UX 최적화**: 
  - **가운데 정렬 보정**: 다양한 모바일 기기 해상도에서도 종료 문구가 완벽하게 중앙에 위치하도록 정렬 로직을 강화하였습니다.
  - **단어 단위 줄바꿈**: `break-keep` 로직을 적용하여 텍스트가 단어 중간에서 끊기지 않고 자연스럽게 줄바꿈되도록 개선하였습니다.
- **성능 및 안정성 고도화**: 
  - **하드웨어 가속**: 안내문구 화면에 `will-change` 및 `GPU 가속`을 적용하여 애니메이션 시 발생하던 드득거림(Stuttering) 현상을 해결하였습니다.
  - **레이아웃 안정화**: 복잡한 필터 효과를 최적화하고 애니메이션 수치를 조정하여 화면 걸침 현상을 수정하였습니다.
